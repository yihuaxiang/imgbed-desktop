# 分发文件选择指南

## 当前构建结果

根据你的构建情况，`dist/` 目录中可能包含以下文件：

### 1. `win-unpacked/` 文件夹 ⚠️ 临时方案

**位置**：`dist/win-unpacked/`

**内容**：
- `Imgbed.exe` - 应用主程序
- `ffmpeg.dll` - 媒体库
- 其他依赖文件

**使用方法**：
1. 将整个 `win-unpacked` 文件夹打包成 ZIP
2. 分发给用户
3. 用户解压后，双击 `Imgbed.exe` 运行

**优点**：
- ✅ 可以立即使用
- ✅ 不需要安装

**缺点**：
- ❌ 需要保持整个文件夹完整
- ❌ 不能只复制 exe 文件
- ❌ 不会创建开始菜单快捷方式

**适用场景**：临时测试、快速分发

---

### 2. 便携版 exe（推荐）⭐

**文件名**：`Imgbed 1.0.0.exe` 或类似

**特点**：
- 单个 exe 文件
- 无需安装，双击即用
- 可以放在任意位置

**使用方法**：
- 直接分发给用户
- 用户双击运行即可

**适用场景**：个人使用、U盘携带、便携需求

**如何生成**：
```bash
npm run build:win:portable
```

---

### 3. 安装程序 exe（推荐用于正式发布）

**文件名**：`Imgbed Setup 1.0.0.exe` 或类似

**特点**：
- 标准的 Windows 安装程序
- 会安装到系统目录
- 创建开始菜单快捷方式
- 可以在"程序和功能"中卸载

**使用方法**：
- 分发给用户
- 用户双击安装程序
- 按照向导完成安装

**适用场景**：正式发布、企业部署、需要系统集成

**如何生成**：
```bash
npm run build:win:x64
```

---

## 推荐方案

### 场景 1：快速测试/临时使用
👉 **使用 `win-unpacked` 文件夹**
- 将整个文件夹打包成 ZIP
- 分发给用户解压使用

### 场景 2：个人使用/便携需求
👉 **使用便携版 exe**（推荐）
- 构建便携版：`npm run build:win:portable`
- 分发生成的 `Imgbed x.x.x.exe` 文件

### 场景 3：正式发布/企业部署
👉 **使用安装程序**
- 构建安装程序：`npm run build:win:x64`
- 分发生成的 `Imgbed Setup x.x.x.exe` 文件

---

## 当前情况

根据你的构建结果，**建议使用 `win-unpacked` 文件夹**：

1. **打包分发**：
   ```bash
   cd dist
   zip -r Imgbed-win-unpacked.zip win-unpacked/
   ```

2. **用户使用**：
   - 解压 ZIP 文件
   - 进入 `win-unpacked` 文件夹
   - 双击 `Imgbed.exe` 运行

---

## 如何生成便携版或安装程序

如果图标问题已解决，可以重新构建：

### 生成便携版（推荐）
```bash
npm run build:win:portable
```
生成文件：`dist/Imgbed 1.0.0.exe`

### 生成安装程序
```bash
npm run build:win:x64
```
生成文件：`dist/Imgbed Setup 1.0.0.exe`

## 文件大小对比

- `win-unpacked/` 文件夹：约 150-200 MB
- 便携版 exe：约 150-200 MB（压缩后）
- 安装程序 exe：约 100-150 MB（压缩后）

## 总结

**当前最佳选择**：使用 `win-unpacked` 文件夹打包成 ZIP 分发

**长期推荐**：
1. 修复图标问题
2. 构建便携版 exe（最方便）
3. 或构建安装程序（最正式）
